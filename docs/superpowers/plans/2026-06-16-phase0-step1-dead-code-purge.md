# Phase 0 · Step 1 — Dead-Code Purge (`ai_sdk` + `trial_utils`) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Delete the two genuinely-dead code clusters in the Gatewayz backend — the never-instantiated `ai_sdk` chat adapter + provider client, and the orphaned `trial_utils` module — with **zero behavior change** and a green test suite.

**Architecture:** Pure deletion + reference cleanup. No new code, no logic change. Each cluster is removed in its own commit after proving (a) nothing in `src/` imports it and (b) the default test suite + lint stay green. This is the safe warm-up that clears noise before the Phase 0 refactors (canonical contract, `chat.py` decomposition).

**Tech Stack:** Python 3.10–3.12, FastAPI, pytest (`-n auto`, `asyncio_mode=auto`, `--timeout=30`, default markers exclude `smoke`/`integration`/`slow`), ruff + black (`-l 100`) + isort.

## Context for the implementer (read first)

This is the first executable step of **Phase 0** of the *Gatewayz One* architecture (`docs/superpowers/specs/2026-06-16-gatewayz-one-architecture-design.md`, §8 / §8.1). A pre-implementation audit (2026-06-16) corrected the spec's original deletion list. **Only two clusters are verified-dead and in scope here:**

1. **`ai_sdk`** — an exported `AISDKChatAdapter` (`src/adapters/chat/ai_sdk.py`) plus a provider client (`src/services/providers/ai_sdk_client.py`). The adapter is exported from `src/adapters/chat/__init__.py` but **never instantiated** anywhere; the client is **not** registered in `PROVIDER_ROUTING`. Its only external reference is one entry in a parametrized import-smoke test.
2. **`trial_utils`** — `src/utils/trial_utils.py`. Its **only** importer in the entire repo is its own test, `tests/utils/test_trial_utils.py`. No production module imports it (verified by grep).

**Explicitly OUT OF SCOPE (do not touch in this plan):**

- `helicone` and `clarifai` — these are *working* (if redundant) provider adapters referenced across ~20 files each (caches, `provider_registry`, `models.py`, `gateway_registry`, health monitors, pricing, `model_transformations`, `connection_pool`, 5 migrations). Per spec §11, redundant working providers are **deactivated via a DB `is_active` flag in Phase 1 (reversible)** — *not* code-deleted. Leave every `helicone`/`clarifai` reference intact.
- `notdiamond`, `code_router`, `anthropic_transformer` — live routing features / shared utility. Keep.
- All live trial code (`partner_trial_service`, `trial_service`, `db/trials.py`, `routes/trial_analytics.py`, `routes/partner_trials.py`, `validate_trial_with_free_model_bypass`, `schemas/trials.py`, `schemas/trial_analytics.py`). The decision on record is **keep partner trials**; only `trial_utils` is dead.

**Why no behavior change is possible here:** both clusters are unreachable from any request path. The `ai_sdk` import-redirect (`src/services/__init__.py` `_RelocatedModuleFinder`) means `import src.services.ai_sdk_client` resolves to `src/services/providers/ai_sdk_client.py`; once the file and its registry entry are gone, that import path must also be removed from the one test that uses it, or the test errors on import.

**Note on `src/gatewayz_backend.egg-info/SOURCES.txt`:** it lists the old paths. It is autogenerated build metadata, regenerated on the next build/install — do **not** hand-edit it.
